# AI Native 工作方式示例

## 服务器监控
- [ ] CPU > 80% 持续 5 分钟 → 钉钉通知 @后端值班
- [ ] 内存 > 90% → 钉钉通知 + 自动扩容
- [ ] 磁盘 > 85% → 钉钉通知 @SRE
- [ ] API 错误率 > 1% → 钉钉通知 + 创建工单
- [ ] 响应时间 P99 > 500ms → 钉钉通知 @性能组

## 业务监控
- [ ] 日活下降 > 10% → 日报提醒 @产品
- [ ] 订单量异常 > 20% → 立即通知 @业务 + @技术
- [ ] 支付成功率 < 95% → 立即通知 @支付组

## 代码质量
- [ ] CI 失败 → Slack 通知 #ci-failures + @作者
- [ ] 覆盖率下降 > 5% → PR 阻塞 + @作者
- [ ] 安全漏洞 (高/严重) → 立即通知 + 工单 + @安全
- [ ] 依赖过期 > 30 天 → 周报汇总 @TechLead

## 财务自由 App 专属
- [ ] 后端服务宕机 → 立即通知 @小贾
- [ ] 数据库连接失败 → 立即通知 @小贾
- [ ] 前端 500 错误 > 10 次/小时 → 通知 @小贾
- [ ] API 响应 > 2s → 通知 @小贾

## AI 自动化规则

1. **告警去重**: 相同告警 10 分钟内只通知一次
2. **告警升级**: 30 分钟未响应 → 升级通知上级
3. **自动恢复**: 指标恢复正常 → 发送恢复通知
4. **周报汇总**: 每周五 17:00 → 发送本周告警汇总
5. **误报学习**: 标记误报 → 自动调整阈值
