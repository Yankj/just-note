## 二、核心价值主张

### 一句话介绍

> **像发消息一样记录一切，AI 自动分类整理，让知识自然生长。**

### 与 flomo 对比

| 维度 | flomo | just-note-ai | 优势 |
|------|-------|-------------|------|
| 输入方式 | 微信/APP | 微信/飞书 | ✅ 更灵活 |
| 整理方式 | 手动标签 | **AI 自动分类** | ✅ **核心差异** |
| 内容类型 | 通用笔记 | **9 类（含收支）** | ✅ 更全面 |
| 数据位置 | 云端 | **本地 + 云端可选** | ✅ 数据自主 |
| 价格 | ¥12/月 | **免费** | ✅ 免费开源 |

### 三大卖点

1. **零摩擦输入** - 发消息即可，不用打开任何界面
2. **AI 自动整理** - 不用手动标签，AI 自动分类
3. **数据自主** - 本地存储，完全掌控

## 五、双语版本问题

### 需要双语吗？

**建议**: **需要！**

**理由**:
1. ✅ **OpenClaw 用户国际化** - 很多英文用户
2. ✅ **技术工具通用** - 代码/命令本身就是英文
3. ✅ **扩大用户群** - 不只局限于中文用户
4. ✅ **提升专业度** - 国际化形象

### 双语实现方案

#### 方案 A: 双语文档（推荐）

```
README.md       # 中文
README.en.md    # 英文
SKILL.md        # 中文
SKILL.en.md     # 英文
```

**优点**:
- 维护简单
- 用户自选语言
- SEO 友好

#### 方案 B: 单一文件双语

```markdown
# 记一下 (just-note-ai)

## 快速开始 / Quick Start

中文内容...

---

English content...
```

**优点**:
- 一个文件
- 对比阅读

**缺点**:
- 文件较长
- 维护稍复杂

#### 方案 C: 自动检测语言

根据用户系统语言自动显示。

**优点**: 用户体验最好  
**缺点**: 实现复杂，不推荐
