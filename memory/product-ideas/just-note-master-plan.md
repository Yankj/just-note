## 一、产品定位

> **像发微信一样记录灵感，AI 自动帮你整理成知识库。**

### 核心洞察

**痛点**: 「打开 Notion 界面、操作一通之后想记录的灵感已经忘了」

**产品哲学**: 从输入开始就要足够简单

| 传统工具 (Notion/IMA) | 记一下 |
|---------------------|--------|
| 打开应用 → 新建文档 → 选择模板 → 编辑 | 直接输入（消息/语音） |
| 用户手动分类、打标签 | AI 自动归类、关联 |
| 用户设计文档结构 | AI 自动排版、组织 |
| 以「文档」为中心 | 以「想法」为中心 |

### 一句话定位

**像发微信一样记录灵感，AI 自动帮你整理成知识库。**

| **OpenClaw Skill** | 极客版/验证版 | 开发者、技术爱好者 | ✅ MVP 完成 |
| **独立 App** | 大众版/商业化版 | 普通知识工作者 | ⏳ 等待验证 |

**协同效应**:
- OpenClaw 验证核心假设 → 降低独立 App 风险
- OpenClaw 用户可迁移到独立 App → 早期种子用户
- 技术逻辑可复用 → 降低开发成本

## 四、技术架构

### 4.1 OpenClaw 版本架构

```
微信/飞书消息 → OpenClaw Gateway → just-note Skill → 文件持久化
                                           ↓
                                    memory/just-note/
                                           ↓
                                    AI 分类/标签/检索
```

### 4.2 独立 App 架构（规划）

```
微信/飞书/网页 → API Gateway → 后端服务 → PostgreSQL
                                     ↓
                                  AI 服务
                                     ↓
                                  检索引擎 (Elasticsearch)
```

### 4.3 持久化方案对比

| 维度 | 文件持久化 | 数据库持久化 |
|------|-----------|-------------|
| 写入速度 | ✅ 快 | ✅ 快 |
| 检索速度 | ⚠️ 慢（需索引） | ✅ 快（原生索引） |
| 人类可读性 | ✅ 直接用文本编辑器 | ❌ 需专用工具 |
| AI 理解难度 | ✅ 低（Markdown 语义化） | ⚠️ 高（需理解 schema） |
| 版本控制 | ✅ 天然支持 Git | ❌ 需额外设计 |
| 并发写入 | ❌ 文件锁问题 | ✅ 事务支持 |

**结论**: 对于「灵感记录」场景（写多读少），**文件持久化更优**，但可加数据库索引层加速检索。

## 五、内容类型设计

### 9 种内容类型

| 类型 | 英文标识 | 说明 | 示例 |
|------|---------|------|------|
| 灵感 | inspiration | 突然的想法 | 「做一个 AI 笔记工具」 |
| 想法 | idea | 成型的思路 | 「可以用 OpenClaw 验证」 |
| 知识 | knowledge | 学到的知识点 | 「文件持久化更适合写多读少场景」 |
| 支出 | expense | 消费记录 | 「午餐 ¥35」 |
| 收入 | income | 收入记录 | 「稿费 ¥500」 |
| 日记 | diary | 日常记录 | 「今天完成了 MVP」 |
| 任务 | task | 待办事项 | 「下周找朋友试用」 |
| 引用 | quote | 名言/摘录 | 「从输入开始就要足够简单」 |
| 其他 | other | 无法分类的 | 任意内容 |

### 笔记格式标准

```markdown
---
title: "AI 生成的标题"
type: inspiration
created: 2026-03-28T00:57:00+08:00
day-id: 2026-03-28
tags: [tag1, tag2, tag3]
---

# 标题

## 原始内容
用户输入的内容...

## AI 整理
- 分类：inspiration
- 标签：tag1, tag2, tag3
- 关联：[相关笔记链接]
```

## 六、CLI 命令设计

### 记录类

| 命令 | 说明 | 示例 |
|------|------|------|
| `write --type <type> --content "内容"` | 明确参数记录 | `just-note write --type inspiration --content "想法"` |
| `quick "内容"` | 快速记录（type=other） | `just-note quick "突然想到的"` |

### 查询类

| 命令 | 说明 | 示例 |
|------|------|------|
| `today` | 查看今日记录 | `just-note today` |
| `yesterday` | 查看昨日记录 | `just-note yesterday` |
| `list` | 列出所有记录 | `just-note list` |
| `list --type <type>` | 按类型筛选 | `just-note list --type expense` |
| `list --from <date>` | 按日期范围 | `just-note list --from 2026-03-01` |
| `search "<keyword>"` | 关键词搜索 | `just-note search "产品"` |
| `diary --date <date>` | 日记视图（时间线） | `just-note diary --date 2026-03-28` |

### 统计类

| 命令 | 说明 | 示例 |
|------|------|------|
| `stats` | 总体统计 | `just-note stats` |
| `stats --type <type>` | 按类型统计 | `just-note stats --type expense` |
| `weekly` | 本周总结 | `just-note weekly` |
| `monthly` | 本月总结 | `just-note monthly` |
| `daily-summary` | AI 生成每日总结 | `just-note daily-summary` |
| `weekly-ai` | AI 生成本周总结 | `just-note weekly-ai` |
| `monthly-ai` | AI 生成本月总结 | `just-note monthly-ai` |

### 导出类

| 命令 | 说明 | 示例 |
|------|------|------|
| `export --format <fmt>` | 导出 | `just-note export --format obsidian` |

## 八、关键假设与验证

### 需要验证的核心假设

1. **输入摩擦是核心痛点**
   - 验证方法：记录「想记但没记」的时刻
   - 成功标准：用户反馈「确实比打开 Notion 简单」

2. **用户信任 AI 自动分类**
   - 验证方法：观察用户是否手动修正分类
   - 成功标准：修正率 < 20%

3. **用户会主动检索历史记录**
   - 验证方法：追踪检索频率
   - 成功标准：每周检索 ≥ 2 次

4. **极客用户愿意为数据主权付费**
   - 验证方法：询问付费意愿
   - 成功标准：> 30% 表示愿意付费
