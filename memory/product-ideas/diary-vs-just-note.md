# 日记框架 vs 闪记：对比分析

**创建时间**: 2026-03-26  
**目的**: 对比已有日记设计和新闪记思路，找出差异和整合方案

---

## 一、已有日记框架（现有）

### 1.1 目录结构

```
memory/
├── agent-diary/          # AI 助理的工作日记
│   └── 2026/
│       └── YYYY-MM-DD.md
└── user-diary/           # 用户的个人日记
    └── 2026/
        └── YYYY-MM-DD.md
```

### 1.2 设计特点

| 维度 | 已有设计 |
|------|---------|
| **定位** | 日记（diary） |
| **内容类型** | 日记为主 |
| **输入方式** | 通过对话让 AI 记录 |
| **整理方式** | AI 按日期组织 |
| **检索方式** | 按日期查找 |
| **格式** | 自由格式（未定义） |
| **Skill** | 无独立 Skill，通过对话 |

### 1.3 使用场景

**agent-diary**:
- AI 助理每天的工作记录
- 任务完成情况
- 与用户的交互记录

**user-diary**:
- 用户的个人日记
- 生活工作记录
- 反思和感悟

## 三、关键差异对比

### 3.1 核心理念

| 维度 | 日记框架 | 闪记 |
|------|---------|------|
| **记录粒度** | 按天（每天一篇） | 按条（每条独立） |
| **内容组织** | 时间线 | 类型 + 时间线 |
| **输入场景** | 主动打开对话 | 发消息（被动接收） |
| **摩擦度** | 中（需打开界面） | 低（发消息即可） |
| **AI 参与** | 辅助写作 | 自动分类 + 标签 |

### 3.2 使用场景对比

| 场景 | 日记框架 | 闪记 |
|------|---------|------|
| 灵感来了 | ❌ 不适合 | ✅ 完美 |
| 记录支出 | ❌ 不适合 | ✅ 完美 |
| 写日记 | ✅ 适合 | ✅ 也适合 |
| 记录知识点 | ❌ 不适合 | ✅ 完美 |
| 每日复盘 | ✅ 适合 | ⚠️ 需要汇总功能 |
| 检索历史 | ⚠️ 按日期找 | ✅ 关键词 + 类型 |

### 3.3 功能对比

| 功能 | 日记框架 | 闪记 |
|------|---------|------|
| 快速输入 | ❌ | ✅ |
| AI 自动分类 | ❌ | ✅ |
| AI 标签 | ❌ | ✅ |
| 收支统计 | ❌ | ✅ |
| 每日汇总 | ❌ | ✅（计划中） |
| 语义检索 | ❌ | ✅（计划中） |
| 知识关联 | ❌ | ✅（计划中） |

## 四、整合方案

### 4.1 方案 A: 闪记包含日记

**思路**: 日记作为闪记的一个内容类型

```
闪记 (just-note)
├── 灵感
├── 想法
├── 知识
├── 收支
├── 日记  ← 日记是其中一种类型
├── 任务
└── 引用
```

**优点**:
- ✅ 统一入口，用户体验一致
- ✅ 所有功能复用（检索/标签/关联）
- ✅ 日记可以和其他内容关联

**缺点**:
- ⚠️ 日记的「连续性」被打破（变成独立条目）
- ⚠️ 每日复盘需要额外汇总

**实现**:
```markdown
---
title: "2026-03-26 日记"
type: diary
date: 2026-03-26
tags: [diary, daily]
---

# 2026-03-26 日记

## 今日概览
- 灵感：3 条
- 支出：¥200
- 任务：完成 5 项

## 日记正文
今天...
```

---

### 4.2 方案 B: 日记和闪记并存

**思路**: 两个独立 Skill，服务不同场景

```
日记 (daily-diary)     闪记 (just-note)
├── 按天组织            ├── 按条记录
├── 长文本              ├── 短文本
├── 反思/复盘           ├── 灵感/想法/知识/收支
└── 每日一篇            └── 随时记录
```

**优点**:
- ✅ 定位清晰，不混淆
- ✅ 各自优化（日记重连续性，闪记重快速）
- ✅ 用户可以分开使用

**缺点**:
- ⚠️ 两个入口，用户需要选择
- ⚠️ 功能重复（都要检索/标签）
- ⚠️ 数据分散

**实现**:
- 日记 Skill：`daily-diary` — 写日记专用
- 闪记 Skill：`just-note` — 快速记录一切

---

### 4.3 方案 C: 统一底层，不同视图

**思路**: 同一套数据，不同呈现方式

```
底层存储：memory/flash-memo/YYYY-MM-DD-HHMM.md

日记视图：按天聚合（类似 daily-diary）
闪记视图：按条展示（类似 just-note）
```

**优点**:
- ✅ 数据统一，不重复
- ✅ 满足不同需求（连续性 vs 快速）
- ✅ 可以互相转换

**缺点**:
- ⚠️ 实现复杂度高
- ⚠️ 需要额外的聚合逻辑

**实现**:
```bash
# 日记视图：聚合某一天的所有记录
just-note diary --date 2026-03-26

# 闪记视图：按类型筛选
just-note list --type inspiration
```

## 五、我的推荐

### 推荐方案：**方案 C（统一底层，不同视图）**

**理由**:
1. **数据统一** — 不会分散在两个地方
2. **灵活性高** — 可以按天看（日记），也可以按类型看（闪记）
3. **扩展性好** — 未来可以加更多视图（周报/月报/主题）
4. **符合 flomo 理念** — flomo 也是「记录一切」，但可以按天回顾

### 实现路径

**阶段 1（MVP）**: 先做闪记（just-note）
- 支持 9 类内容
- 每条独立文件
- AI 自动分类

**阶段 2（日记视图）**: 增加日记功能
- 按天聚合记录
- 支持写长日记
- 每日汇总推送

**阶段 3（复盘功能）**: 增加周报/月报
- AI 自动生成
- 统计图表

## 六、对闪记设计的输入

### 6.1 从日记框架学到的

1. **日期组织重要** — 用户习惯按日期回顾
2. **连续性价值** — 日记的价值在于长期坚持
3. **复盘需求** — 用户需要定期回顾（每日/每周/每月）

### 6.2 闪记需要增加的

1. **每日汇总** — 每天结束时推送今日记录汇总
2. **日记模式** — 支持写长日记（不只是短记录）
3. **时间线视图** — 按时间线展示所有记录
4. **周报/月报** — AI 自动生成的周期性总结

### 6.3 笔记格式调整

```markdown
---
title: "AI 生成的标题"
type: inspiration  # 9 类之一
created: 2026-03-26T12:00:00+08:00
day-id: 2026-03-26  # 用于按天聚合
tags: [tag1, tag2, tag3]
---

# 标题

## 原始内容
...

## AI 整理
...
```
