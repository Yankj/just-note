### [22:22] OpenClaw vs Claude Code 工作流设计

**记录时间**: 2026-03-18 22:22

**类型**：工具协作架构

**思考内容**：

**背景**：
- OpenClaw 生态多样：云端、本地、toB、toC
- Claude Code 之前用得少，主要用类似 SR、公司 Casa 等软件
- 最近深度使用 Claude Code 后有新认知

**核心定位**：
1. Claude Code = 专业开发工程师（专注编程开发）
2. OpenClaw = 电脑管家/大管家（综合系统管理）

**协作工作流**：
```
用户 (手机端)
    │
    ▼
OpenClaw (管家)
    │
    ▼
Claude Code (工程师 - 电脑端/云端)
    │
    ▼
写代码/交付
```

**Skill 分工**：
- Claude Code：编程开发、需求交付相关 skills
- OpenClaw：更综合、更系统的电脑使用相关 skills

**价值**：
1. 这个认知可以支撑未来高效开发工具/工作流/工作模式的探索
2. 提供信息支撑，帮助搭建高校开发的工具工作流
